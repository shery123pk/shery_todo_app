# üöÄ How to Start the Project Manually

## Prerequisites

Make sure you have installed:
- Node.js 18+ and npm
- Python 3.11+ and UV package manager
- PostgreSQL database (or Neon account)

---

## Step 1: Start Backend API

Open **Terminal/Command Prompt #1**:

```bash
# Navigate to backend folder
cd backend

# Activate virtual environment (if using venv) or just run with uv
# Create .env file first if you haven't (copy from .env.example)

# Start the backend server
uv run uvicorn app.main:app --reload --port 8001

# You should see:
# INFO:     Uvicorn running on http://127.0.0.1:8001
# INFO:     Application startup complete
```

**Backend will be running at:** http://localhost:8001

**Test it:**
```bash
curl http://localhost:8001/health
# Should return: {"status":"healthy","service":"project-management-api"}
```

**API Docs:** http://localhost:8001/docs

---

## Step 2: Start Frontend

Open **Terminal/Command Prompt #2** (new window):

```bash
# Navigate to frontend folder
cd frontend

# Install dependencies (first time only)
npm install

# Start development server
npm run dev

# You should see:
# ‚ñ≤ Next.js 15.x
# - Local:        http://localhost:3002
# ‚úì Ready in 2.5s
```

**Frontend will be running at:** http://localhost:3002

---

## Step 3: Access the Application

Open your browser and go to:
```
http://localhost:3002
```

**First Time Setup:**
1. Click "Sign Up" to create an account
2. Fill in your details (email, password, name)
3. After signup, you'll be redirected to dashboard
4. Start creating tasks!

## üêõ Troubleshooting

### Backend won't start
```bash
# Check if port 8001 is already in use
# On Windows:
netstat -ano | findstr :8001

# Kill the process using the port
taskkill /PID <process_id> /F

# Then restart backend
```

### Frontend won't start
```bash
# Check if port 3002 is in use
netstat -ano | findstr :3002

# Kill the process
taskkill /PID <process_id> /F

# Clear Next.js cache and restart
rm -rf .next
npm run dev
```

### Database connection error
- Verify your `DATABASE_URL` is correct in backend `.env`
- Make sure Neon database is active (not suspended)
- Test connection: `uv run python -c "from app.database import engine; print('Connected!')"`

### OpenAI API error
- Verify your `OPENAI_API_KEY` is valid
- Check you have credits remaining
- Test: `curl https://api.openai.com/v1/models -H "Authorization: Bearer YOUR_KEY"`
